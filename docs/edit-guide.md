# プリントワールド — 静的HTML 編集手順書

納品後、御社側でテキスト・画像・リンクを差し替える際の参考資料です。  
レイアウトやレスポンシブの調整は `mobile.css` を触る必要がある場合がありますが、**通常の文言・画像の変更は HTML のみ**で完結します。

## 2. 基本的な編集方法

### テキストを変える

1. 該当ページの HTML をテキストエディタで開く  
2. 画面上の文言を検索（Ctrl+F）  
3. `<p>` や `<h2>` などタグ内の文字列を書き換え  
4. 保存してブラウザで再読み込み（キャッシュが残る場合は Ctrl+Shift+R）

### 画像を差し替える

1. 新しい画像を `images/` にアップロード  
2. HTML 内の `src="images/ファイル名.png"` を新ファイル名に変更  
   - または **同じファイル名で上書き**（HTML の変更は不要）

### リンクを設定する

```html
<a href="https://example.com/">表示テキスト</a>
```

- サイト内ページ: `href="guide-flow.html"` のように相対パス  
- 電話: `href="tel:0566785885"`  
- 未設定の場合は `href="#"` のまま（`docs/link-inventory.csv` を参照）

## 5. 触らない方がよいもの

- **`mobile.css` の末尾付近** — 最終上書き用の `@media` ブロック。不用意な変更で全幅のレイアウトが崩れます  
- **`app.js` のメニュー定義** — ハンバーガーメニューのリンク一覧（必要なら `app.js` 174–180行付近）  
- **CSS 変数**（`--sec9-card-s` 等）— デザイン比率のスケール用

レイアウトが崩れた場合のみ、該当セクションの class 名を開発担当者に共有してください。

## 6. ローカルでの確認方法

1. フォルダをそのまま開く、または簡易サーバーで表示  
2. 幅の確認: ブラウザの開発者ツール（F12）→ デバイスモード  
   - 推奨: **320 / 768 / 1024 / 1920** px  
3. 変更後は GitHub に push すると Vercel プレビューが更新されます

## 7. Maker Town 移行時の注意

本番（`print-world.jp`）では HTML の一部が Maker Town のタグに置き換わります。

- **商品一覧・詳細** — 静的な `.plp-card` / `.pdp-*` は MT 商品ゾーンに差し替え  
- **カテゴリ・タグリンク** — MT のカテゴリURLに変更  
- **ガイド・固定ページ** — 静的HTMLのままアップロード可能な場合が多い

移行作業は別途実施します。静的版の編集は Vercel プレビュー上で先行して問題ありません。
