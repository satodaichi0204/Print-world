# メーカータウン 商品登録 手順書（デザイン可能にする5ステップ）

テスト商品（パープル01）で確立した「デザインツールが動く商品」を作る手順。
全23商品はこの型の繰り返し＋CSV一括で登録する。

## 前提：先にマスタを用意（1回だけ・使い回し）

- **アイテムオプション値マスタ** … カラー（各色＋RGB）とサイズを登録。全商品で使い回せる。
- **加工位置マスタ** … 印刷位置（例：左胸、前面中央、背面など）。
- **加工機マスタ** … 印刷機（例：プリンター）。加工可能拠点＋優先順位を設定。
- **加工方法マスタ** … 「加工機×加工位置」の組み合わせ（例：左胸_インクジェット）。DPI・加工形状SVG・加工方法コード・**加工価格**を設定。

## 5ステップ（1商品ごと）

### 1. アイテム基本情報
- 代表画像・名称・表示順・納期・最低数量・**アイテムコード**（一意）。説明・素材・サイズHTML・配送コメント。

### 2. アイテム個体（色×サイズ＝SKU）
- 商品ページに出す**全カラー×全サイズ分**を登録。
- 各行：カラー（オプション値）・サイズ・表示可否・ベンダー名・型番・カラー名・カラーコード・サイズ名・**JAN（一意）**・価格。

### 3. 位置別アイテム画像
- 「位置別アイテム画像追加」→ 表示名・表示順・表示可否・位置別アイテム画像コード（一意）。
- **カラーごとにサムネイル画像**（＝デザインツールの下地）をアップロード。必要ならマスク画像。

### 4. 加工方法指定・アイテムエリア関連 ★最重要
- 加工機リストで加工機を選択→追加→**「有効」にチェック**。
- 加工方法リストで加工方法を選択→追加。表示名・表示順・加工方法指定コードを入力。
- **キャンバス画像**を登録→保存→**座標設定**（プリント範囲をドラッグ）。
- **右側「位置別アイテム画像」欄で、Step3の位置別画像を選択→追加→その座標も設定**。
  - ← ここが抜けると `data-color-count=0 / data-print-type-count=0` になり、デザインボタンが動かない。

### 5. 在庫
- Product→在庫→在庫追加：**JAN・拠点・数量**を各SKU分登録。

## 動作確認（合格基準）
- フロント商品ページの「デザインする」ボタンのHTMLで、
  `data-color-count ≥ 1` かつ `data-print-type-count ≥ 1` になっていること。
- 「デザインする」→（複数色なら）カラー選択モーダル→デザインツール起動→カート追加まで通ること。

## 下地画像（デザインツール用）の仕様
- 形式：**PNG（背景透過）**
- 向き：**正面（front）**。※加工位置が背面等なら該当面も。
- 内容：**Tシャツ単体（人物・小物なし）**、中央配置、余白を全色で統一。
- サイズ：**1000×1000px 以上を推奨**。**同一商品の全色で同じサイズ・同じ配置**（座標がずれないため）。
- 解像度：加工方法のDPI（現状200）を満たすこと。
- ファイル名：**半角英数字のみ**（CSVインポート時の必須条件）。

## CSV一括登録（量産時）
- アイテムマスタ一覧・アイテム個体・アイテムオプション値マスタに「CSV操作」あり。
- **手順**：管理画面で各CSVを一度ダウンロード→ヘッダ行を保ったまま編集→アップロード。
- 画像はZIP（ファイル名は半角英数字）でCSVと同時アップロード可。
- ID空欄＝新規作成、ID記入＝既存編集。
- ※正確なカラム構成は、実機でCSVをダウンロードして確定する。
